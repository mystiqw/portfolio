03 — My Approach

How I think

UX is not styling — it's problem solving. Three principles guide my work.

01
Understand before designing
I first ask: who uses this, when, and in what context? Psychological understanding comes before the first frame.
02
Structure before aesthetics
Good information architecture makes an interface usable. Visual beauty amplifies that — but never replaces it.
03
Hand off developer-ready
Designs have to be buildable. My CS background helps me create specs that actually work in code.
